DNS Server Configuration

The main BIND configuration file is /etc/named.conf. Here's an overview of key settings:

  • listen-on port 53 { 127.0.0.1; }; - Defines where the server listens for requests.
  • directory "/var/named"; - Specifies the directory for zone files.
  • allow-query { localhost; }; - Restricts which clients can query the DNS server.
  • dnssec-enable yes; - Enables DNSSEC for security.

Zone Files

Zone files contain DNS record definitions. Define your forward and reverse zones in /etc/named.conf:

zone "example.com" {
    type master;
    file "example.com.db";
};

Ensure zone files are stored in the directory defined in the configuration.

Useful Commands

Use these commands to manage BIND and check DNS resolutions:

  • dig @localhost example.com - Test DNS resolution.
  • rndc reload - Reload the DNS server.
  • named-checkzone - Validate zone files.

RNDC Utility

RNDC is used to control the BIND server. To generate a key for RNDC:

rndc-confgen -r /dev/urandom -a

Ensure to include the RNDC key in /etc/named.conf for proper integration.
