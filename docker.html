Docker Overview

Docker is a container management system designed to make it easier to create, deploy, and run applications using containers. Here are some key points about Docker:

  • Containers are lightweight and run in a loosely isolated environment without needing a hypervisor.
  • Docker engine is a client-server application with a daemon server, REST API, and a command-line interface.
  • Docker images are built from a series of layers representing instructions in a Dockerfile.
  • Containers create a writable layer on top of the underlying image layers. The writeable layer stores any changes, and it is deleted when the container is removed.

Common Docker Commands

Here is a list of useful Docker commands:

  • Docker attach: Access a Docker machine.
  • Docker build: Build an image from a Dockerfile.
  • Docker exec: Run a command in a running container.
  • Docker images: List images.
  • Docker info: Display system-wide information.
  • Docker inspect: Return low-level information about Docker objects.
  • Docker logs: Fetch logs from a container.
  • Docker network: Manage networks.
  • Docker node: Manage swarm nodes.
  • Docker ps: List containers.
  • Docker pull: Pull an image or repository from a registry.
  • Docker push: Push an image or repository to a registry.
  • Docker restart: Restart one or more containers.
  • Docker rm: Remove one or more containers.
  • Docker rmi: Remove one or more images.
  • Docker run: Run a command in a new container.
  • Docker start: Start one or more stopped containers.
  • Docker stop: Stop one or more running containers.
  • Docker swarm: Manage Swarm clusters.
  • Docker volume: Manage volumes.

Building and Running Docker Containers

mkdir -p docker/images
cd docker/images
vi Dockerfile

An example Dockerfile to create a static site:

FROM nginx
WORKDIR /usr/share/nginx/html
COPY index.html ./

vi index.html:

add html content and than you can build a docker container using nginx:

To build and run a Docker container with this Dockerfile:

Docker build . -t la/static:latest
Docker run -d --name=static-site -p 80:80 la/static:latest

To view running containers, use:

Docker ps
